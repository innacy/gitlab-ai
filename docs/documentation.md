# gitlab-ai — AI-Powered GitLab CLI Tool

An interactive command-line tool that integrates GitLab with AI-powered code review via [Cursor Background Agents](https://docs.cursor.com/en/background-agent/api/overview). Review merge requests, manage issues, and post feedback — all from a chatbot-style terminal session.

## Prerequisites

- **Go 1.24.6+** installed ([download](https://go.dev/dl/))
- A **GitLab** account on `gitlab.widas.de` (or another instance)
- A `~/.netrc` file with GitLab credentials
- A **Cursor API key** (`CURSOR_API_KEY` environment variable)

---

## Installation

### Build from source

```bash
git clone <repository-url>
cd cli
make build
```

The binary is placed in `./bin/gitlab-ai`.

### Cross-compilation

```bash
make build-linux     # Linux (amd64)
make build-darwin    # macOS (Intel + Apple Silicon)
make build-windows   # Windows (amd64)
make build-all       # All platforms
```

## Authentication

gitlab-ai reads your GitLab token from `~/.netrc`. Create or edit the file:

```
machine gitlab.widas.de
login your-username
password glpat-xxxxxxxxxxxxxxxxxxxx
```

Set proper file permissions:

```bash
chmod 600 ~/.netrc
```

The tool looks for the `.netrc` entry matching the configured GitLab host (`gitlab.widas.de`) and uses the `password` field as the API token.

## Configuration

### Config file

```
~/.gitlab-ai/config.yaml
```

### Minimal setup

```yaml
gitlab:
  base_url: "https://gitlab.widas.de"
  api_version: "v4"
  default_project: ""
```

### AI (Cursor) setup

Set the Cursor API key as an environment variable:

```bash
export CURSOR_API_KEY="your-cursor-api-key"
```

The tool uses the [Cursor Background Agent API](https://docs.cursor.com/en/background-agent/api/overview) via [cursor-go-sdk](https://github.com/unkn0wncode/cursor-go-sdk) for AI-powered reviews.

### Environment variable overrides

Any config key can be overridden using the `GITLAB_AI_` prefix:

```bash
export GITLAB_AI_GITLAB_BASE_URL="https://gitlab.widas.de"
export GITLAB_AI_CLI_VERBOSE=true
```

## Interactive Commands

Run the tool to enter interactive mode:

```bash
./bin/gitlab-ai
```

```
🤖 gitlab-ai — Interactive Mode

Commands:
  start                    Start GitLab session
  mr <number> -p <project> Review MR with AI
  add comment <number>     Post review as MR comment
  tickets -p <project>     List project issues & save
  exit                     End session

Auto-exit after 1 minute of inactivity.

gitlab-ai>
```

The session **auto-exits after 1 minute of inactivity**.

---

### `start`

Authenticate with GitLab and begin a session.

```
gitlab-ai> start
```

**What happens:**
1. Reads credentials from `~/.netrc` for the configured GitLab host
2. Authenticates via the GitLab API
3. Displays user info and confirms the connection

**Output:**
```
✓ Authenticated as John Doe (@johndoe)
✓ GitLab: https://gitlab.widas.de
✓ Session started — ready for commands
```

---

### `mr <number> -p <project>`

Fetch a merge request and generate an AI-powered code review using Cursor.

```
gitlab-ai> mr 226 -p mgmt
```

**What happens:**
1. Fetches MR #226 from project `mgmt` (metadata + full diff)
2. Sends the diff to Cursor Background Agent for AI review
3. Polls until the agent completes (up to 3 minutes)
4. Displays the structured review in the terminal
5. Saves the review to `mgmt-226.md` (replaces if file exists)
6. Stores the review context for `add comment`

**Output:**
```
✓ MR fetched: "Fix authentication flow"
✓ Changes: 5 files, +120 -45 lines
✓ AI review generated

# MR Review: Fix authentication flow (#226)
...

✓ Review saved to: mgmt-226.md
```

**File naming:** `{project}-{mr_number}.md` — e.g., `mgmt-226.md`
If the file already exists, it is replaced.

---

### `add comment <number>`

Post the most recent AI review as a comment on a GitLab merge request.

```
gitlab-ai> add comment 226
```

**What happens:**
1. Takes the review generated by the last `mr` command
2. Posts it as a note/comment on MR #226
3. Uses the same project from the last `mr` review

**Output:**
```
✓ Review posted to MR #226
✓ View at: https://gitlab.widas.de/mgmt/-/merge_requests/226#note_12345
```

**Prerequisite:** You must run `mr <number> -p <project>` first to generate a review.

---

### `tickets -p <project>`

List all open issues assigned to you in a project and save them to a markdown file.

```
gitlab-ai> tickets -p issues
```

**What happens:**
1. Fetches all open issues assigned to the authenticated user
2. Displays them in a formatted table
3. Saves the full list to a markdown file

**Output:**
```
Issues for project 'issues' (Assigned to you)

┌─────┬──────────────────────────────────┬──────────┬──────────┬────────────┐
│ ID  │ Title                            │ State    │ Labels   │ Updated    │
├─────┼──────────────────────────────────┼──────────┼──────────┼────────────┤
│ 45  │ Fix authentication bug           │ opened   │ bug      │ 2 days ago │
│ 52  │ Add user profile endpoint        │ opened   │ feature  │ 1 week ago │
└─────┴──────────────────────────────────┴──────────┴──────────┴────────────┘

Total: 2 issues
✓ Tickets saved to: issues_2026-03-03_14-30.md
```

**File naming:** `{project}_{date}_{time}.md` — e.g., `issues_2026-03-03_14-30.md`

## Workflows

### Standard MR Review

```
gitlab-ai> start
gitlab-ai> mr 226 -p mgmt
gitlab-ai> add comment 226
gitlab-ai> exit
```

### Review Multiple MRs

```
gitlab-ai> start
gitlab-ai> mr 226 -p mgmt
gitlab-ai> add comment 226
gitlab-ai> mr 42 -p frontend
gitlab-ai> add comment 42
gitlab-ai> exit
```

> **Note:** `add comment` always posts the **most recent** review. Review and post one at a time.

### Daily Issue Triage

```
gitlab-ai> start
gitlab-ai> tickets -p mgmt
gitlab-ai> tickets -p frontend
gitlab-ai> exit
```

Output files are saved automatically with timestamps — one per project per session.

### Review-Only (No Post)

```
gitlab-ai> start
gitlab-ai> mr 226 -p mgmt
# Review is saved to mgmt-226.md — don't run add comment
gitlab-ai> exit
```

### Key Components

| Component | Package | Description |
|-----------|---------|-------------|
| **REPL** | `cmd/repl.go` | Interactive chatbot loop with 1-minute timeout, command dispatch |
| **GitLab Client** | `pkg/gitlab/` | Wraps [go-gitlab](https://github.com/xanzy/go-gitlab) — auth, MR fetch, comment posting, issue listing |
| **Cursor AI** | `cmd/repl.go` | Integrates [cursor-go-sdk](https://github.com/unkn0wncode/cursor-go-sdk) — launches background agent, polls for completion, extracts review from conversation |
| **Prompt Builder** | `pkg/ai/prompts.go` | Constructs structured review prompts with MR metadata, diff, and section instructions |
| **Config** | `pkg/config/` | Viper-based YAML config with env var overrides and validation |
| **Output** | `pkg/output/` | Terminal formatting (colors, tables, icons) and markdown file generation |

### Data Flow: MR Review

```
User types: mr 226 -p mgmt
        │
        ▼
  cmd/repl.go → handleMR()
        │
        ├── glClient.GetMergeRequest("mgmt", 226)
        │       ├── FindProject()         → GET /projects/:id
        │       ├── GetMergeRequest()     → GET /projects/:id/merge_requests/226
        │       └── GetMergeRequestChanges() → GET .../changes
        │
        ├── ai.BuildSystemPrompt() + ai.BuildReviewPrompt()
        │
        ├── cursorCli.LaunchAgent(prompt, source)
        │       ├── Poll cursorCli.GetAgent() every 3s
        │       └── cursorCli.GetConversation() → extract review
        │
        ├── output.PrintReview()          → terminal display
        │
        ├── os.WriteFile("mgmt-226.md")   → save markdown
        │
        └── store lastReview              → for add comment
```

## Configuration Reference

### Full config example (`~/.gitlab-ai/config.yaml`)

```yaml
gitlab:
  base_url: "https://gitlab.widas.de"
  api_version: "v4"
  default_project: ""

ai:
  provider: "openai"
  openai:
    api_key_env: "OPENAI_API_KEY"
    model: "gpt-4-turbo"
    max_tokens: 4000
    temperature: 0.7
  anthropic:
    api_key_env: "ANTHROPIC_API_KEY"
    model: "claude-3-opus-20240229"
    max_tokens: 4000
    temperature: 0.7

review:
  template:
    sections:
      - name: "Summary"
        prompt: "Provide a brief summary of the changes"
        required: true
      - name: "Code Quality"
        prompt: "Assess code quality, readability, and maintainability"
        required: true
      - name: "Security Concerns"
        prompt: "Identify any security vulnerabilities or risks"
        required: true
      - name: "Performance Impact"
        prompt: "Evaluate potential performance implications"
        required: false
      - name: "Testing Coverage"
        prompt: "Comment on test coverage and quality"
        required: true
      - name: "Suggestions"
        prompt: "Provide actionable improvement suggestions"
        required: true
      - name: "Approval Recommendation"
        prompt: "Recommend approve, request changes, or reject with reasoning"
        required: true
  output:
    directory: "./reviews"
    filename_pattern: "{project}_{mr_number}.md"
    include_metadata: true
    include_diff: false
  filters:
    exclude_files:
      - "*.lock"
      - "vendor/*"
      - "*.generated.go"

issues:
  output:
    directory: "./issues"
    filename_pattern: "{project}_issues_{date}.md"
  fields:
    - "id"
    - "title"
    - "description"
    - "state"
    - "labels"
    - "assignee"

cli:
  color_output: true
  markdown_rendering: true
  verbose: false
  confirm_before_post: true
```

## Troubleshooting

### Authentication

| Error | Cause | Fix |
|-------|-------|-----|
| `no GitLab token found` | No `.netrc` entry for the host | Add entry to `~/.netrc` |
| `.netrc file has permissions ...` | File permissions too open | `chmod 600 ~/.netrc` |
| `authentication failed — invalid token` | Token expired/revoked or wrong URL | Regenerate token, check `base_url` |

**Verify token manually:**
```bash
curl -H "PRIVATE-TOKEN: glpat-xxx" https://gitlab.widas.de/api/v4/user
```

**Generate a new token:**
`https://gitlab.widas.de/-/user_settings/personal_access_tokens` (requires `api` scope)

### Session

| Error | Cause | Fix |
|-------|-------|-----|
| `No active session` | Haven't run `start` | Type `start` first |
| `Session timed out` | 1 minute with no input | Run the tool again |
| `Session already active` | Already started | Use `exit` first, then `start` |

### MR Review

| Error | Cause | Fix |
|-------|-------|-----|
| `CURSOR_API_KEY not set` | Missing environment variable | `export CURSOR_API_KEY="..."` |
| `Failed to fetch MR` | Wrong MR number or project path | Check number and use full project path |
| `Cursor agent failed` | Agent couldn't process the request | Check Cursor API status, retry |
| `Cursor agent timed out` | Agent took >3 minutes | Retry; large diffs may take longer |

### Tickets

| Error | Cause | Fix |
|-------|-------|-----|
| `Failed to fetch tickets` | Wrong project path | Use the exact GitLab project path |
| `Failed to save tickets` | Permission denied | Check directory write permissions |

### Quick Fixes

```bash
# Fix .netrc permissions
chmod 600 ~/.netrc

# Set Cursor API key
export CURSOR_API_KEY="your-key"

# Verify GitLab connection
curl -H "PRIVATE-TOKEN: glpat-xxx" https://gitlab.widas.de/api/v4/user

# Rebuild after changes
make build
```
